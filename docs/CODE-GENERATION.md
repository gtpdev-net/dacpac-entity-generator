# Code Generation Pipeline

## Overview

DataManager generates C# Entity Framework Core code from SQL Server schema metadata. The pipeline produces:

1. **Entity classes** — One `.cs` file per table/view with Data Annotations
2. **SQL Server configuration** — `ModelBuilder` configuration per database
3. **SQLite configuration** — SQLite-compatible configuration variant
4. **SQL Server DbContext** — `SQLDbContext` with all DbSets and configuration registration
5. **SQLite DbContext** — `SQLiteDbContext` for testing/local development
6. **Discovery reports** — JSON and HTML reports of stored procedures, triggers, sequences

## Pipeline Modes

### Mode 1: Database-Driven (Web UI)

Uses `DataManagerGenerationOrchestrator` with `DataManagerDbSchemaDataSource`.

```
DataManagerDb (SQL Server)
  └── ISchemaDataSource.GetTablesForGenerationAsync()
      └── Filter: IsActive=true, IsSelectedForLoad=true, PersistenceType ≠ 'D'
          └── Map SourceColumn → ColumnDefinition
              └── Generate code
```

Triggered from the **EF Generation** page in the Blazor UI. Real-time log streaming via `BlazorGenerationLogger` events.

### Mode 2: File-Driven (Legacy Console)

Uses `GenerationOrchestrator` with Excel + DACPAC files.

```
Excel (.xlsx)                    DACPAC files (.dacpac)
  └── Read & filter rows           └── Extract model.xml (ZIP)
      └── Group by Server/DB           └── Parse XML (XDocument)
          └── Match Excel rows to DACPAC schema
              └── Generate code
```

## Pipeline Stages

### Stage 1: Load Table Definitions

**Database mode:** Queries `SourceTable` and `SourceColumn` entities filtered by `IsSelectedForLoad = true` and `PersistenceType ≠ 'D'`, then maps them to `TableDefinition` / `ColumnDefinition` objects.

**File mode:**
1. Locate Excel file in the input directory
2. Read and filter rows where `Generate = true`
3. Group by Server/Database
4. For each group, extract `model.xml` from the corresponding DACPAC
5. Parse the XML to build `TableDefinition` objects, retaining only columns listed in Excel

### Stage 2: Validate Tables

Each `TableDefinition` is validated by `EntityClassGenerator.ValidateEntityClass()`:

- Table name must produce a valid C# class name via `NameConverter.ToPascalCase()`
- All column names must produce valid C# property names
- Namespace must be non-empty

Tables failing validation are skipped with an error logged.

### Stage 3: Generate Entity Classes

`EntityClassGenerator.GenerateEntityClass()` produces a `.cs` file per table:

```csharp
// <auto-generated />
using System;
using System.ComponentModel.DataAnnotations;
using System.ComponentModel.DataAnnotations.Schema;
using DataLayer.Core.Entities;

namespace DataLayer.Core.Entities.{Server}.{Database}
{
    [Table("{TableName}", Schema = "{Database}")]
    public class {ClassName} : BaseEntity
    {
        // Properties...
    }
}
```

#### Property Generation Rules

| Scenario | Generated Code |
|---|---|
| Primary key (single) | `[Key]` attribute |
| Composite key | No `[Key]` — configured in `ModelBuilder` |
| Non-nullable without default | `[Required]` attribute |
| String with max length | `[MaxLength(n)]` attribute |
| Computed column | `[DatabaseGenerated(DatabaseGeneratedOption.Computed)]` |
| Row version | `[Timestamp]` attribute |
| Column name mapping | `[Column("sql_column_name")]` always applied |

#### Backing-Field Pattern for Default Values

Non-nullable value-type properties with SQL defaults use a backing-field pattern to prevent EF Core sentinel value warnings:

**Bool with default:**
```csharp
// SQL Default: ((1))
[Column("is_active")]
public bool IsActive
{
    get => _isActive ?? true;   // Returns TRUE when null (database default)
    set => _isActive = value;
}
private bool? _isActive;
```

**Int with default:**
```csharp
// SQL Default: ((0))
[Column("retry_count")]
public int RetryCount
{
    get => _retryCount ?? 0;   // Returns 0 when null (database default)
    set => _retryCount = value;
}
private int? _retryCount;
```

**DateTime with zero default** (invalid for datetime2):
```csharp
// SQL Default: ((0))
[Column("created_date")]
public DateTime CreatedDate
{
    get => _createdDate ?? DateTime.MinValue;
    set => _createdDate = value;
}
private DateTime? _createdDate;
```

#### Property Name Conflict Resolution

If a column's PascalCase name matches the class name, the class is suffixed with `Entity`:

```csharp
// Table "Status" with column "status" → class "StatusEntity"
public class StatusEntity : BaseEntity { ... }
```

### Stage 4: Generate EF Core Configuration

`EntityConfigurationGenerator` produces configuration classes per Server/Database pair.

#### SQL Server Configuration

```csharp
namespace DataLayer.Core.Configuration.{Server}.{Database}
{
    public static class {Database}EntityConfiguration
    {
        public static void Configure(ModelBuilder modelBuilder)
        {
            modelBuilder.Entity<ClassName>();
            // Indexes, check constraints, unique constraints, column configs...
        }
    }
}
```

Configuration includes:
- **Indexes** — `HasIndex()` with `IsUnique()`, `HasFilter()`, `HasDatabaseName()`
- **Check constraints** — `HasCheckConstraint()`
- **Unique constraints** — `HasAlternateKey()` with `HasName()`
- **Backing fields** — `HasField("_backingField")`
- **Decimal precision** — `HasColumnType("decimal(p,s)")`
- **Collation** — `UseCollation()`
- **Computed columns** — `HasComputedColumnSql()` with optional `stored: true`
- **Default values** — `HasDefaultValueSql()`

#### SQLite Configuration

A separate SQLite-compatible variant is generated with these differences:

| SQL Server | SQLite |
|---|---|
| `ToTable("name", Schema = "schema")` | `ToTable("database_name")` (no schema) |
| `HasCheckConstraint()` | Omitted |
| `UseCollation()` | Omitted |
| `HasFilter()` on indexes | Omitted |
| `HasComputedColumnSql()` | Omitted |
| `HasColumnType("decimal(p,s)")` | `HasColumnType("REAL")` |
| `HasDefaultValueSql("(getdate())")` | `HasDefaultValueSql("(datetime('now','localtime'))")` |
| `HasDefaultValueSql("(newid())")` | `HasDefaultValueSql("(lower(hex(randomblob(16))))")` |

### Stage 5: Generate DbContext

`DbContextGenerator` produces both SQL Server and SQLite DbContext files:

- **`SQLDbContext`** — References SQL Server configuration classes
- **`SQLiteDbContext`** — References SQLite configuration classes

Both inherit from `BaseDbContext` and contain identical DbSet properties.

### Stage 6: File Output

`FileWriterService` manages the output file system:

- Cleans stale generated directories before writing
- Creates server/database directory structure
- Writes entity files to `{outputDir}/{Server}/{Database}/{TableName}.cs`
- Writes configuration files to `{outputDir}/{Server}/{Database}/{Database}EntityConfiguration.cs`
- Tags generated files with `// <auto-generated />` header

### Stage 7: Discovery Reports (Optional)

`ReportWriterService` produces reports of discovered schema elements:

- **JSON reports** — Machine-readable per-database reports
- **HTML reports** — Human-readable formatted reports

Reports include stored procedures, triggers, sequences, and other elements that may require manual attention.

## Name Conversion

`NameConverter` handles SQL→C# identifier transformation:

- **PascalCase conversion** — `user_name` → `UserName`, `first-name` → `FirstName`
- **Identifier sanitization** — Strips non-alphanumeric characters, prefixes digits with `_`
- **C# keyword escaping** — Prefixes reserved words with `@` (e.g., `@class`, `@event`)

## Hash-Based DACPAC Deduplication

`HashHelper.Sha256Hex()` computes a SHA-256 hash of the `model.xml` content. This hash is stored in `SourceDatabase.LastImportedModelHash` and compared on subsequent imports. If the hash matches, the import is skipped entirely, avoiding unnecessary database operations.
